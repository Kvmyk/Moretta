Installation

Deployment Steps

Step 1: Prep
Hardware
- **8 GB RAM** minimum.
- **Docker** and **Compose** installed.
- Linux (Ubuntu/Debian) or Windows/macOS.
Step 2: Environment
Secrets
- Copy `.env.example` to `.env`.
- Set `VAULT_ENCRYPTION_KEY`.
- Add your AI Provider API Keys.
# 1. Clone the repository
git clone https://github.com/Kvmyk/moretta.git
cd moretta

# 2. Prepare environment
cp .env.example .env
nano .env

# 3. Startup (Linux/macOS)
chmod +x start.sh
./start.sh

# 3. Startup (Windows)
start.bat

Identity

Initial User Setup

For maximum security, the repository contains no pre-provisioned accounts. You must create your first administrator via the master console.

Action 1
Login Master
Go to **localhost:8080/auth**. Login with `admin` / `admin123` (or as configured in `.env`).
Action 2
Create Primary User
Switch to **moretta** realm. Go to **Users** -> **Add user**. Set a password in the **Credentials** tab.

Reference

Configuration (.env)

Variable Default Role
VAULT_ENCRYPTION_KEY (none) Mandatory. 32-64 hex char key for SQLite vault encryption.
LOCAL_MODEL phi4-mini The Ollama model used for local PII detection.
DEFAULT_PROVIDER claude Main AI engine (claude, openai, gemini, ollama).
SSO_ENABLED true Toggles OIDC validation requirement for all API requests.
